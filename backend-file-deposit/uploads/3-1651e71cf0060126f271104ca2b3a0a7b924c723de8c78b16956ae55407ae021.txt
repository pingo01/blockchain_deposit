基于区块链技术的数字资产存证系统
总体描述： 探索区块链技术的应用，构建一个用于重要电子文件（如电子合同、版权证书）哈希值存证的系统。项目组不需要实现完整的区块链，但需理解并模拟区块链的核心概念，实现数据上链（模拟）、存证查询与验证等功能。
    【分析与设计重点】：分析文件上传者、验证者等角色用例；设计“存证记录”、“区块”、“链”等核心领域模型，清晰表达其不可篡改的特性。在架构设计上，需设计一个模拟的区块链服务层，该层应提供生成区块、构建哈希链、验证数据完整性等核心接口，并在整体架构中明确其位置。
    【实现与测试要求】：实现文件上传生成哈希、将存证信息（模拟）写入“区块”、以及基于链式结构验证文件是否被篡改的核心流程；测试存证与验证流程的正确性，并演示对篡改文件的识别能力。

开发工具可选，鼓励使用新技术栈，如：
前端：Vue.js / React / Angular (任一现代框架) 或 Bootstrap。
后端：Spring Boot (Java) / Express.js (Node.js) / Django (Python) / .NET Core (C#)。
数据库：MySQL / MongoDB。
文档工具：Microsoft Office / WPS Office / Markdown
部署：鼓励使用Docker容器化，并可部署至云平台（如阿里云、腾讯云、Heroku）的试用套餐

系统分为四个模块：1. 用户认证模块	处理用户身份管理与权限控制	用户注册、登录、角色校验、账号状态管理	边界类：用户登录界面、注册界面；控制类：用户认证控制器
2. 文件处理与哈希模块	负责文件上传、哈希生成核心逻辑	文件格式校验、文件大小限制、SHA256 哈希值计算	边界类：文件上传界面、文件验证界面；控制类：文件处理控制器
3. 区块链模拟核心模块	实现区块生成、哈希链构建（核心）	新区块创建、区块哈希计算、哈希链维护、存证记录写入区块	控制类：区块生成控制器、链管理控制器；实体类：Block、HashChain
4. 存证查询与验证模块	处理存证记录查询与文件完整性验证	按存证编号 / 文件信息查询、待验证文件哈希对比、篡改识别	边界类：查询界面、验证结果展示界面；控制类：查询验证控制器

每个模块的功能：
模块 1：用户认证模块
用例名称	角色	优先级	前置条件	后置条件
用户注册	潜在用户	中	无	生成新用户账号，状态为 “正常”
用户登录	已注册用户	高	用户已注册	建立会话，返回角色对应的操作界面
密码重置	已注册用户	中	用户已注册且忘记密码	密码更新为新值，可重新登录
修改个人信息	已注册用户	中	用户已注册且处于登录状态	个人信息（如昵称、手机号、头像等）更新生效


模块 2：文件处理与哈希模块
用例名称	角色	优先级	前置条件	后置条件
文件上传	文件上传者	高	用户已登录，角色为上传者	文件暂存至服务器，返回文件元数据
文件格式校验	系统自动触发	高	文件开始上传	确认文件是否符合格式 / 大小要求
哈希值生成	系统自动触发	高	文件校验通过	生成文件唯一哈希值（SHA256）


模块 3：区块链模拟核心模块
用例名称	角色	优先级	前置条件	后置条件
存证记录创建	系统	高	文件哈希值已生成	生成存证记录（含哈希值、用户 ID 等）
区块生成	系统	高	存证记录已创建	生成新区块（含上一区块哈希）
哈希链维护	系统	高	新区块已生成	区块加入哈希链，更新链尾指针

模块 4：存证查询与验证模块
用例名称	角色	优先级	前置条件	后置条件
按存证编号查询	上传者、验证者	高	用户已登录	返回对应存证记录详情（哈希值、区块信息等）
按文件名查询	文件上传者	中	用户已登录，角色为上传者	返回该用户名下匹配文件名的存证记录
文件验证	验证者	高	用户已登录	返回验证结果（未篡改 / 已篡改 / 无存证）
